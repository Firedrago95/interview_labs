# 🎟 쿠폰 선착순 발급 시스템 (대용량 트래픽 처리)

## 📌 개요
단기간에 엄청난 트래픽이 몰리는 상황에서 시스템이 다운되지 않고 정확하게 N명에게만 혜택을 제공하는 능력을 평가합니다.

## 🎯 요구사항
1. **선착순 발급 API**
   - 특정 쿠폰을 1,000장 한정으로 선착순 발급합니다.
   - 10만 명의 유저가 동시에 발급 요청을 보내는 상황을 가정합니다.
2. **중복 발급 방지**
   - 1명의 유저는 동일한 쿠폰을 1장만 발급받을 수 있습니다.

## 💡 평가 포인트 (리뷰 기준)
- RDBMS만 사용했을 때의 한계(Lock 경합 병목)를 이해하고, **Redis**를 활용해 원자적(Atomic)으로 재고를 차감할 수 있는가? (예: Lua Script 또는 `INCR`/`DECR`)
- 쿠폰 발급 성공 여부를 빠르게 응답하고, 실제 DB 저장은 비동기(Kafka 등)로 지연 처리(Eventual Consistency)하여 부하를 분산시킬 수 있는가?
- Rate Limiting 기법을 이해하고 있는가?
