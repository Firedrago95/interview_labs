# 🛒 재고 차감 및 주문 시스템 (Spring Boot 동시성)

## 📌 개요
E-commerce의 핵심인 주문 처리 시 발생하는 동시성(Concurrency) 문제를 어떻게 제어하는지 평가합니다.

## 🎯 요구사항
1. **상품 주문 API**
   - 사용자 ID, 상품 ID, 구매 수량을 입력받아 주문을 생성합니다.
   - 상품의 현재 재고를 확인하고, 구매 수량만큼 재고를 차감합니다.
2. **동시성 제어 (핵심)**
   - 여러 사용자가 동시에 같은 상품을 구매하려 할 때, 재고가 음수(-)가 되거나 업데이트가 유실(Lost Update)되지 않아야 합니다.

## 💡 평가 포인트 (리뷰 기준)
- 비관적 락(Pessimistic Lock)과 낙관적 락(Optimistic Lock)의 차이를 이해하고 적절한 방식을 선택했는가?
- 트랜잭션의 범위(`@Transactional`)가 적절히 설정되었는가? 데드락 가능성은 없는가?
- 데이터베이스 레벨의 방어 로직(`CHECK (stock >= 0)`)을 고려했는가?

## 배운점
1. `@MappedSuperclass` JPA에서 그냥 상속하면 별도 전역 테이블생성 방지를 위해 컬럼만 맵핑하기 위한 에너테이션
