# 💳 결제 상태머신 및 멱등성 시스템 (PG사 연동)

## 📌 개요
외부 시스템(PG사)과의 연동 시 발생하는 네트워크 장애, 지연, 타임아웃, 중복 호출 문제를 안전하게 방어하는 능력을 평가합니다.

## 🎯 요구사항
1. **결제 요청 및 승인**
   - 외부 API(PG사)를 호출하여 결제를 승인받고, 내부 DB에 상태(PENDING -> APPROVED)를 업데이트합니다.
2. **멱등성(Idempotency) 보장**
   - 클라이언트가 결제 버튼을 실수로 두 번 누르거나, 네트워크 오류로 재시도하더라도 **결코 두 번 결제되지 않아야** 합니다.
   - PG사에서 보내는 웹훅(Webhook)이 네트워크 이슈로 중복 수신되더라도 한 번만 처리되어야 합니다.

## 💡 평가 포인트 (리뷰 기준)
- 외부 API 호출 로직이 **DB 트랜잭션 내부를 길게 점유하지 않도록** 트랜잭션 경계를 분리했는가?
- `Idempotency Key`(멱등성 키)를 활용하여 중복 요청을 방어하도록 테이블(`UNIQUE KEY` 등) 및 로직이 설계되었는가?
