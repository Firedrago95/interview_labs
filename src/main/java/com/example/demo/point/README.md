# 💰 포인트 원장(Ledger) 시스템

## 📌 개요
잔액 계산의 정확성과 내역의 무결성을 증명하기 위한 아키텍처 역량을 평가합니다.

## 🎯 요구사항
1. **포인트 적립/사용 API**
   - 사용자는 주문 완료 시 포인트를 적립받고, 결제 시 포인트를 사용할 수 있습니다.
   - 단일 '잔액' 컬럼만 업데이트하는 것이 아니라, "왜 잔액이 변했는지" 모든 이력(History)을 장부 형태로 남겨야 합니다.
2. **동시성 고려**
   - 포인트 사용과 적립이 동시에 일어날 때, 최종 잔액이 어긋나면 안 됩니다.

## 💡 평가 포인트 (리뷰 기준)
- `PointLedger` 테이블을 이중 장부(Append-Only) 형태로 설계하여 과거 내역 추적이 가능한가?
- 포인트를 사용할 때, 차감 순서나 만료 기간을 고려할 수 있는 유연한 구조인가?
